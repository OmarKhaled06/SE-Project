# 4.3 Database Schema Documentation — CampusCare

## Overview

CampusCare uses **PostgreSQL** hosted on **Supabase**, managed through the **Prisma ORM** (v6). The schema consists of 6 tables representing users, roles, issues, photos, comments, and notifications.

---

### Relationships Summary

| From | To | Type | Via |
|---|---|---|---|
| User | UserRole | One-to-Many | `UserRole.userId` |
| User | Issue | One-to-Many (reporter) | `Issue.reporterId` |
| User | Issue | One-to-Many (assignee) | `Issue.assigneeId` |
| User | Comment | One-to-Many | `Comment.authorId` |
| User | IssuePhoto | One-to-Many | `IssuePhoto.uploadedBy` |
| User | Notification | One-to-Many | `Notification.userId` |
| Issue | IssuePhoto | One-to-Many | `IssuePhoto.issueId` |
| Issue | Comment | One-to-Many | `Comment.issueId` |
| Issue | Notification | One-to-Many | `Notification.issueId` |

All foreign key relationships use **CASCADE on delete**: deleting a User cascades to their roles, issues, comments, photos, and notifications. Deleting an Issue cascades to its photos, comments, and notifications.

## Table Definitions

### Table: `User`

Stores all registered users of the system.

| Column | Type | Nullable | Default | Constraints |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | PRIMARY KEY |
| `email` | VARCHAR(255) | No | — | UNIQUE |
| `passwordHash` | TEXT | No | — | — |
| `fullName` | VARCHAR(100) | No | — | — |
| `phone` | VARCHAR(20) | Yes | NULL | — |
| `active` | BOOLEAN | No | `true` | — |
| `createdAt` | TIMESTAMP WITH TIME ZONE | No | `now()` | — |
| `updatedAt` | TIMESTAMP WITH TIME ZONE | No | — | Auto-updated by Prisma |

**Indexes:** `email` (unique)

---

### Table: `UserRole`

Implements the many-to-many relationship between users and roles. A user may hold multiple roles.

| Column | Type | Nullable | Default | Constraints |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | PRIMARY KEY |
| `userId` | UUID | No | — | FK → `User.id` ON DELETE CASCADE |
| `role` | Role ENUM | No | — | — |

**Constraints:** `UNIQUE(userId, role)` — a user cannot have the same role twice.

**Enum: `Role`**
```
MEMBER | WORKER | MANAGER | ADMIN
```

---

### Table: `Issue`

The core entity of the system. Represents a submitted maintenance or campus issue.

| Column | Type | Nullable | Default | Constraints |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | PRIMARY KEY |
| `title` | VARCHAR(120) | No | — | — |
| `description` | TEXT | No | — | — |
| `category` | Category ENUM | No | `OTHER` | — |
| `location` | VARCHAR(200) | No | — | — |
| `status` | IssueStatus ENUM | No | `PENDING` | — |
| `priority` | Priority ENUM | No | `MEDIUM` | — |
| `reporterId` | UUID | No | — | FK → `User.id` ON DELETE CASCADE |
| `assigneeId` | UUID | Yes | NULL | FK → `User.id` ON DELETE CASCADE |
| `resolvedAt` | TIMESTAMP WITH TIME ZONE | Yes | NULL | Set when status → RESOLVED |
| `closedAt` | TIMESTAMP WITH TIME ZONE | Yes | NULL | Set when status → CLOSED |
| `createdAt` | TIMESTAMP WITH TIME ZONE | No | `now()` | — |
| `updatedAt` | TIMESTAMP WITH TIME ZONE | No | — | Auto-updated by Prisma |

**Indexes:** `reporterId`, `assigneeId`, `status`

**Enum: `Category`**
```
ELECTRICAL | PLUMBING | HVAC | CLEANING | FURNITURE | SAFETY | IT | OTHER
```

**Enum: `IssueStatus`**
```
PENDING | ASSIGNED | IN_PROGRESS | RESOLVED | CLOSED
```

**Enum: `Priority`**
```
LOW | MEDIUM | HIGH | URGENT
```

---

### Table: `IssuePhoto`

Stores photo attachments linked to issues. Each photo is either a report photo (submitted with the issue) or a completion photo (submitted by the worker as proof of resolution).

| Column | Type | Nullable | Default | Constraints |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | PRIMARY KEY |
| `issueId` | UUID | No | — | FK → `Issue.id` ON DELETE CASCADE |
| `url` | TEXT | No | — | Relative path: `/uploads/<filename>` |
| `kind` | PhotoKind ENUM | No | `REPORT` | — |
| `uploadedBy` | UUID | No | — | FK → `User.id` ON DELETE CASCADE |
| `createdAt` | TIMESTAMP WITH TIME ZONE | No | `now()` | — |

**Enum: `PhotoKind`**
```
REPORT | COMPLETION
```

---

### Table: `Comment`

Work-log comments written on an issue by any authenticated user (typically workers or managers).

| Column | Type | Nullable | Default | Constraints |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | PRIMARY KEY |
| `issueId` | UUID | No | — | FK → `Issue.id` ON DELETE CASCADE |
| `authorId` | UUID | No | — | FK → `User.id` ON DELETE CASCADE |
| `body` | TEXT | No | — | Max 1000 chars (enforced by API) |
| `createdAt` | TIMESTAMP WITH TIME ZONE | No | `now()` | — |

---

### Table: `Notification`

In-app notifications sent to users when issues are created, updated, assigned, or closed.

| Column | Type | Nullable | Default | Constraints |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | PRIMARY KEY |
| `userId` | UUID | No | — | FK → `User.id` ON DELETE CASCADE |
| `title` | TEXT | No | — | Short notification title |
| `body` | TEXT | No | — | Notification message body |
| `issueId` | UUID | Yes | NULL | FK → `Issue.id` ON DELETE CASCADE |
| `read` | BOOLEAN | No | `false` | — |
| `createdAt` | TIMESTAMP WITH TIME ZONE | No | `now()` | — |

```bash

npx prisma migrate deploy   # apply all migrations
npm run seed                 # run the seed script
```

The seed uses `upsert` so it is safe to run multiple times without creating duplicates.
