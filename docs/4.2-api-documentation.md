# 4.2 API Documentation — CampusCare

## Overview

| Item | Value |
|---|---|
| Base URL | `http://localhost:4000` |
| API Prefix | `/api` |
| Data Format | JSON (request & response) |
| Auth Scheme | Bearer JWT (`Authorization: Bearer <token>`) |
| Token Expiry | 7 days |
| Rate Limit | 200 requests / minute per IP |

### Roles

| Role | Description |
|---|---|
| `MEMBER` | Community member — submits and tracks their own issues |
| `WORKER` | Maintenance worker — assigned to and resolves issues |
| `MANAGER` | Facility manager — oversees all issues, assigns workers |
| `ADMIN` | System administrator — full access including user management |

A user may hold multiple roles simultaneously. Role checks are additive (e.g. ADMIN can do everything MANAGER can).

### Common Error Responses

| Status | Meaning | Body |
|---|---|---|
| 400 | Validation error | `{ "error": "...", "issues": [...] }` |
| 401 | Missing or invalid token | `{ "error": "Missing token" }` |
| 403 | Authenticated but insufficient role | `{ "error": "Forbidden" }` |
| 404 | Resource not found | `{ "error": "Not found" }` |
| 409 | Conflict (e.g. duplicate email) | `{ "error": "Email already registered" }` |
| 500 | Internal server error | `{ "error": "Internal server error" }` |

## Auth Endpoints — `/api/auth`

### POST /api/auth/register

Register a new user account. All new accounts receive the `MEMBER` role by default.

**Authentication:** None

**Request Body**

| Field | Type | Required | Constraints |
|---|---|---|---|
| `email` | string | yes | Valid email, max 255 chars |
| `password` | string | yes | Min 6, max 72 chars |
| `fullName` | string | yes | Min 2, max 100 chars |
| `phone` | string | no | Max 20 chars |

**Example Request**
```http
POST /api/auth/register
Content-Type: application/json

{
  "email": "alice@university.edu",
  "password": "SecurePass1!",
  "fullName": "Alice Johnson",
  "phone": "+20 100 123 4567"
}
```

**Response 201 — Success**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "b3d7e2a1-...",
    "email": "alice@university.edu",
    "fullName": "Alice Johnson",
    "phone": "+20 100 123 4567",
    "active": true,
    "roles": ["MEMBER"]
  }
}
```

**Response 409 — Email taken**
```json
{ "error": "Email already registered" }
```

## Issues Endpoints — `/api/issues`

All issue endpoints require a valid Bearer token.

### GET /api/issues

Fetch a list of issues. The result set is automatically scoped by role:
- **MEMBER:** only sees their own submitted issues
- **WORKER:** only sees issues assigned to them
- **MANAGER / ADMIN:** sees all issues; supports optional `status` query filter

**Authentication:** Bearer token required

**Roles:** Any authenticated user (result scope varies by role)

**Query Parameters**

| Param | Type | Description |
|---|---|---|
| `status` | string | Filter by status: `PENDING`, `ASSIGNED`, `IN_PROGRESS`, `RESOLVED`, `CLOSED` (MANAGER/ADMIN only) |

**Example Request**
```http
GET /api/issues?status=PENDING
Authorization: Bearer <token>
```

**Response 200**
```json
{
  "issues": [
    {
      "id": "issue-uuid-...",
      "title": "Broken light in Library 2F",
      "description": "The overhead light by the window flickers and then turns off.",
      "category": "ELECTRICAL",
      "location": "Library, Floor 2, Aisle B",
      "status": "PENDING",
      "priority": "MEDIUM",
      "reporterId": "user-uuid-...",
      "assigneeId": null,
      "createdAt": "2026-05-10T09:30:00.000Z",
      "updatedAt": "2026-05-10T09:30:00.000Z",
      "resolvedAt": null,
      "closedAt": null,
      "reporter": { "id": "...", "fullName": "Mia Member", "email": "member@campuscare.dev" },
      "assignee": null,
      "_count": { "comments": 0, "photos": 1 }
    }
  ]
}
```

### POST /api/issues

Submit a new maintenance issue.

**Authentication:** Bearer token required

**Roles:** Any authenticated user

**Request Body**

| Field | Type | Required | Constraints |
|---|---|---|---|
| `title` | string | yes | Min 3, max 120 chars |
| `description` | string | yes | Min 10, max 2000 chars |
| `location` | string | yes | Min 2, max 200 chars |
| `category` | string | yes | `ELECTRICAL`, `PLUMBING`, `HVAC`, `CLEANING`, `FURNITURE`, `SAFETY`, `IT`, `OTHER` |
| `priority` | string | no | `LOW`, `MEDIUM` (default), `HIGH`, `URGENT` |

**Example Request**
```http
POST /api/issues
Authorization: Bearer <token>
Content-Type: application/json

{
  "title": "Water leak in Men's Restroom B2",
  "description": "A pipe under the second sink is dripping continuously, causing a puddle on the floor.",
  "location": "Building B, Floor 2, Men's Restroom",
  "category": "PLUMBING",
  "priority": "HIGH"
}
```

**Response 201**
```json
{
  "issue": {
    "id": "new-issue-uuid-...",
    "title": "Water leak in Men's Restroom B2",
    "description": "A pipe under the second sink is dripping continuously, causing a puddle on the floor.",
    "category": "PLUMBING",

    "priority": "HIGH",
```

### PUT /api/issues/:id/status

Update the status of an issue.

**Authentication:** Bearer token required

**Roles:** WORKER, MANAGER, ADMIN

**Path Parameters:** `id` — Issue UUID

**Request Body**

| Field | Type | Required | Allowed Values |
|---|---|---|---|
| `status` | string | yes | `PENDING`, `ASSIGNED`, `IN_PROGRESS`, `RESOLVED`, `CLOSED` |

**Example Request**
```http
PUT /api/issues/issue-uuid-.../status
Authorization: Bearer <token>
Content-Type: application/json

{ "status": "IN_PROGRESS" }
```

**Response 200**
```json
{
  "issue": {
    "id": "issue-uuid-...",
    "status": "IN_PROGRESS",
    "updatedAt": "2026-05-16T10:00:00.000Z"
  }
}
```

> Side effect: A notification is sent to the issue reporter when status changes.

### PUT /api/issues/:id/assign

Assign or unassign a worker to an issue. Automatically sets status to `ASSIGNED` when a worker is assigned, or `PENDING` when unassigned.

**Authentication:** Bearer token required

**Roles:** MANAGER, ADMIN

**Path Parameters:** `id` — Issue UUID

**Request Body**

| Field | Type | Required | Description |
|---|---|---|---|
| `assigneeId` | string (UUID) or null | yes | Worker user ID, or `null` to unassign |

**Example Request**
```http
PUT /api/issues/issue-uuid-.../assign
Authorization: Bearer <token>
Content-Type: application/json

{ "assigneeId": "worker-uuid-..." }
```

**Response 200**
```json
{
  "issue": {
    "id": "issue-uuid-...",
    "assigneeId": "worker-uuid-...",
    "status": "ASSIGNED",
    "updatedAt": "2026-05-16T10:05:00.000Z"
  }
}
```

> Side effect: A notification is sent to the assigned worker.

### POST /api/issues/:id/photo

Upload a photo attached to an issue (report photo or completion proof).

**Authentication:** Bearer token required

**Roles:** Any authenticated user

**Path Parameters:** `id` — Issue UUID

**Request:** `multipart/form-data`

| Field | Type | Required | Description |
|---|---|---|---|
| `photo` | file | yes | Image file (JPEG, PNG) |
| `kind` | string | no | `REPORT` (default) or `COMPLETION` |

**Example Request**
```http
POST /api/issues/issue-uuid-.../photo
Authorization: Bearer <token>
Content-Type: multipart/form-data

photo: [binary image data]
kind: COMPLETION
```

**Response 201**
```json
{
  "photo": {
    "id": "photo-uuid-...",
    "issueId": "issue-uuid-...",
    "url": "/uploads/1715334600000-photo.jpg",
    "kind": "COMPLETION",
    "uploadedBy": "worker-uuid-...",
    "createdAt": "2026-05-16T14:00:00.000Z"
  }
}
```

> Photos are served as static files at `GET /uploads/<filename>`. Prepend the server base URL to reconstruct the full URL.

### PUT /api/admin/users/:id/status

Activate or deactivate a user account. Deactivated users cannot log in.

**Authentication:** Bearer token required

**Roles:** ADMIN only

**Path Parameters:** `id` — User UUID

**Request Body**

| Field | Type | Required | Description |
|---|---|---|---|
| `active` | boolean | yes | `true` to activate, `false` to deactivate |

**Example Request**
```http
PUT /api/admin/users/user-uuid-.../status
Authorization: Bearer <token>
Content-Type: application/json

{ "active": false }
```

**Response 200**
```json
{
  "user": {
    "id": "user-uuid-...",
    "active": false,
    "updatedAt": "2026-05-16T10:00:00.000Z"
  }
}
```

---

### PUT /api/admin/users/:id/role

Grant or revoke a role from a user. Multiple roles can be managed by calling this endpoint once per role.

**Authentication:** Bearer token required

**Roles:** ADMIN only

**Path Parameters:** `id` — User UUID

**Request Body**

| Field | Type | Required | Description |
|---|---|---|---|
| `role` | string | yes | `MEMBER`, `WORKER`, `MANAGER`, or `ADMIN` |
| `on` | boolean | yes | `true` to grant, `false` to revoke |

**Example Request — Grant WORKER role**
```http
PUT /api/admin/users/user-uuid-.../role
Authorization: Bearer <token>
Content-Type: application/json

{ "role": "WORKER", "on": true }
```

**Response 200**
```json
{ "ok": true }
```
