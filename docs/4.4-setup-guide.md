# 4.4 Setup & Installation Guide — CampusCare

## Prerequisites

Ensure the following are installed on your machine before proceeding.

| Tool | Minimum Version | How to Check |
|---|---|---|
| Node.js | 18.x or higher | `node --version` |
| npm | 9.x or higher | `npm --version` |
| Git | Any recent version | `git --version` |
| Expo CLI | (installed via npm) | `npx expo --version` |
| Expo Go app | Latest | Install on iOS / Android from the App Store / Google Play |

> **Database:** CampusCare uses **Supabase** (hosted PostgreSQL) — no local PostgreSQL installation is required. You will need a free Supabase account.

## Part 1 — Backend Setup

### Step 1: Install Backend Dependencies

```bash
cd backend
npm install
```

### Step 2: Configure Environment Variables

Create a `.env` file inside the `backend/` folder:

```bash
# backend/.env

# Supabase connection strings (get from Supabase → Project Settings → Database)
DATABASE_URL="postgresql://postgres.<project-ref>:<password>@aws-0-<region>.pooler.supabase.com:6543/postgres?pgbouncer=true"
DIRECT_URL="postgresql://postgres.<project-ref>:<password>@aws-0-<region>.pooler.supabase.com:5432/postgres"

# JWT secret — use any long random string (e.g. 64 random characters)
JWT_SECRET="replace_with_a_long_random_secret_string"

# Token expiry (optional, defaults to 7d)
JWT_EXPIRES_IN="7d"

# Server port (optional, defaults to 4000)
PORT=4000
```

> **How to get Supabase connection strings:**
> 1. Go to [supabase.com](https://supabase.com) and sign in
> 2. Open your project → **Project Settings** → **Database**
> 3. Scroll to **Connection string** and copy the **URI** for both the pooler (port 6543) and direct connection (port 5432)
> 4. Replace `[YOUR-PASSWORD]` with your database password

### Step 3: Generate Prisma Client

```bash
cd backend
npx prisma generate
```

### Step 4: Apply Database Migrations

```bash
npx prisma migrate deploy
```

This creates all tables in your Supabase database.

### Step 5: Seed the Database

Populate the database with test accounts and a sample issue:

```bash
npm run seed
```

After seeding, the following test accounts are available (all use password `Password1!`):

| Role | Email |
|---|---|
| ADMIN | `admin@campuscare.dev` |
| MANAGER | `manager@campuscare.dev` |
| WORKER | `worker@campuscare.dev` |
| MEMBER | `member@campuscare.dev` |

### Step 6: Run the Backend Server

**Development mode** (auto-reloads on file changes):
```bash
npm run dev
```

**Production mode:**
```bash
npm run build
npm start
```

The API server starts at `http://localhost:4000`. Verify it is running:
```bash
curl http://localhost:4000/health
# Expected: {"ok":true}
```

## Part 2 — Frontend Setup

### Step 1: Install Frontend Dependencies

```bash
cd frontend
npm install
```

### Step 2: Configure the Backend URL

Open [frontend/src/services/api.js](../frontend/src/services/api.js) and update `BASE_URL` to point to your backend server.

```js
// For Android Emulator:
export const BASE_URL = 'http://10.0.2.2:4000';

// For iOS Simulator:
export const BASE_URL = 'http://localhost:4000';

// For a real phone (must be on the same Wi-Fi as your dev machine):
export const BASE_URL = 'http://<YOUR_LAN_IP>:4000';
```

> **Finding your LAN IP:**
> - **Windows:** Run `ipconfig` in Command Prompt → look for IPv4 Address
> - **macOS/Linux:** Run `ifconfig` → look for `inet` under your Wi-Fi adapter

### Step 3: Start the Expo Development Server

```bash
cd frontend
npx expo start -c
```

The `-c` flag clears the Metro bundler cache (recommended after dependency changes).

### Step 4: Open the App on Your Device

After the server starts, a QR code is displayed in the terminal.

| Platform | How to open |
|---|---|
| **Android** | Open the **Expo Go** app → tap **Scan QR Code** |
| **iOS** | Open the **Camera** app → point at the QR code → tap the notification |
| **Android Emulator** | Press `a` in the terminal |
| **iOS Simulator** | Press `i` in the terminal (macOS only, requires Xcode) |

## Part 3 — Third-Party Services

### Supabase (Database)

CampusCare requires a Supabase project for its PostgreSQL database.

1. Create a free account at [supabase.com](https://supabase.com)
2. Create a new project and choose a region close to your users
3. Wait for the project to initialize (~2 minutes)
4. Go to **Project Settings → Database** and copy the connection strings into `backend/.env`
5. Go to **Project Settings → General** to find your project reference ID if needed

> **Important:** If you are on the Supabase free tier, projects pause after 1 week of inactivity. Go to the Supabase dashboard and click **Resume project** if you see database connection errors.

### File Uploads (Local Storage)

Photo uploads (issue report photos and completion photos) are stored locally on the backend server in the `backend/uploads/` folder. This folder is created automatically when the first photo is uploaded.

Uploaded files are served as static assets at:
```
http://localhost:4000/uploads/<filename>
```

> For production deployments, replace local storage with a cloud storage service (e.g. Supabase Storage, AWS S3, or Cloudinary) by updating `backend/src/middleware/upload.ts`.

## Common Issues & Fixes

| Problem | Likely Cause | Fix |
|---|---|---|
| `Cannot find module 'expo-secure-store'` | Missing dependency | Run `npx expo install expo-secure-store` inside `frontend/` |
| `Login failed` / 401 error | Seed not run or wrong credentials | Run `npm run seed` in `backend/` |
| `Network request failed` | Wrong BASE_URL or backend not running | Check `BASE_URL` in api.js matches your LAN IP and port 4000; confirm backend is running |
| Database connection timeout | Supabase project is paused | Resume the project from the Supabase dashboard |
| `prisma generate` fails | Prisma client out of sync | Delete `backend/node_modules/.prisma` and re-run `npx prisma generate` |
| QR code scan fails | Phone and laptop on different networks | Ensure both are on the same Wi-Fi network (hotspot or router) |

## Environment Variables Reference

### Backend (`backend/.env`)

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | Supabase pooled connection string (port 6543) |
| `DIRECT_URL` | Yes | Supabase direct connection string (port 5432), used by Prisma migrate |
| `JWT_SECRET` | Yes | Secret key for signing JWT tokens — keep private |
| `JWT_EXPIRES_IN` | No | Token lifetime, default `7d` |
| `PORT` | No | HTTP port the server listens on, default `4000` |
| `CORS_ORIGIN` | No | Allowed CORS origin, default `*` (allow all) |
